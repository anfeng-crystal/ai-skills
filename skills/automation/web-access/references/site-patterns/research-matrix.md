# Research Matrix Pattern

## 适用信号
- 任务要求“最新”“当前”“今天”“最近”
- 需要多来源交叉验证、比对口径或找冲突点
- 需要同时看官方源、直接当事方和第三方资料

## 首选入口
- 优先用直接来源、官方入口和已知站点路径
- 宿主搜索只有在明确可用时才辅助使用
- 如果宿主搜索不稳但需要可编程搜索，且 `BRAVE_SEARCH_API_KEY` 已配置，可临时启用 `scripts/brave-search.mjs`
- 只有来源页面本身依赖动态渲染时，才补浏览器/CDP
- 对时间敏感问题，优先选直接来源和最近日期

## 最小步骤
1. 先列来源矩阵：官方源、直接当事方、独立来源。
2. 能直接访问来源页时，先访问来源页，不先走宿主搜索。
3. 每条关键事实至少确认一次直接来源；存在冲突时保留冲突，不强行统一。
4. 记录绝对日期、发布时间和访问时间。
5. 输出时区分：已证实事实、来源冲突、待确认问题。

## 证据记录
- 来源名称和链接
- 发布日期、事件日期、访问日期
- 是否直接来源
- 与其他来源的冲突点

## 风险 / 停问
- 只有单一来源时，不把结论写满
- 日期模糊时，不用“今天/最近”代替绝对日期
- 宿主搜索结果与直接来源冲突时，以直接来源为主，并把冲突写清楚
- 无法联网时，明确只能给访问方案或离线背景，不能冒充最新信息
