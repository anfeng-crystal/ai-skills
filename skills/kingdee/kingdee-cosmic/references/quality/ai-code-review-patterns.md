# 苍穹 Java 自检规则参考

## 定位
- 本文件用于苍穹 Java 代码审查、自检和质量检查。
- 只作为风险检查清单；是否修改代码取决于用户指令、当前任务类型和项目规则。
- 不进行批量修复，不写经验库，不替代编译、测试、联调或官方资料校验。

## 逻辑与边界
- 查询结果使用前必须判空，重点检查 `ORM`、`BusinessDataServiceHelper.loadSingle`、`DynamicObject.get`、`Map.get` 和方法参数。
- 集合访问前必须判空和判空集合，禁止未检查就调用 `get(0)`。
- 字段值类型转换必须按元数据类型和项目现有写法处理，禁止凭经验强转。
- 循环边界、分页偏移、字符串截取、日期区间必须覆盖边界值。
- 状态分支必须覆盖未知状态；字符串比较优先使用常量在前的 `equals`。

## 安全与性能
- 密钥、密码、连接串、token 必须来自配置或环境，不得硬编码到代码。
- 涉及查询条件时优先使用 ORM、`QFilter`、服务助手和参数化能力，禁止拼接用户输入。
- 流、文件句柄、锁和外部连接必须明确释放；优先使用 try-with-resources。
- 禁止循环内逐条查询数据库；先收集 key，再批量查询并构建 Map。
- 可能返回大数据集的查询必须加业务过滤、分页或上限。

## 苍穹平台
- 插件事件、基类和注册方式必须与插件类型匹配。
- 后续使用的字段必须在合适的字段准备阶段声明或确保加载。
- 保存、删除、提交、审核等动作优先使用平台标准服务，不绕开平台机制。
- 实体标识、字段标识、分录标识必须来自元数据、项目代码或可靠资料。
- 跨库、跨对象、外部 API、MQ、通知等动作必须检查事务边界和补偿路径。

## 常见错误模式
| 编号 | 模式 | 检查点 |
|---|---|---|
| ERR-001 | 查询结果未判空 | 访问查询结果字段前必须处理空结果 |
| ERR-002 | 循环内查询 | 改为批量查询和本地映射 |
| ERR-003 | 保存 API 形态错误 | 按当前依赖签名确认 `SaveServiceHelper` 入参 |

## 注释与说明
- 新增类、工具类、公共方法、抽取出的业务方法和复杂私有方法必须写功能性注释。
- 注释只说明职责、业务口径、平台约束、边界条件和长期有效但不直观的原因。
- 不把实施过程、排查路径、验证经过、沟通记录或交付口径写入代码注释、skills、操作说明或示例说明。
