# Data Analyst

> **Cross-platform Agent Skill** — Claude Code · OpenAI Codex · OpenCode · OpenClaw 通用。
> 跨平台 SKILL.md，遵循开放 Agent Skill 规范。

## 触发边界
- 用户要分析 SQL 查询结果、CSV、Excel、TSV 或多份表格数据，并输出结论、图表、报告或分析型工作簿时使用。
- 重点处理指标定义、数据体检、分析路径、图表选择、结论表述和结果校验。
- 只做简单表格编辑、格式微调或纯文件转换时，不单独触发；需要工作簿产物时复用 `Excel`。

## 核心原则
- 先明确业务问题，再处理数据；不要先画图后找结论。
- 先做数据体检，再做指标计算；口径未清前不输出确定性判断。
- 默认走最轻路径：能直接回答就不强制产出文件，用户明确要 `.xlsx`、图表包或交付件时再升级到 `Excel`。
- 结论必须能回溯到字段、筛选条件、时间范围和样本范围。
- 不把缺失值、重复值、异常值和口径冲突当成“数据已清洗完毕”。

## 快速工作流
1. 明确分析问题：
   - 目标指标是什么
   - 时间范围、维度、分组口径是什么
   - 交付形式是直接回答、Markdown 报告还是 `.xlsx`
2. 判断数据入口：
   - SQL 查询或数据库结果
   - CSV / TSV
   - Excel 工作簿
   - 多源合并
3. 先做数据体检（按标准判断是否通过）：
   - **字段类型**：字段类型是否合理（日期字段是否可解析、金额字段是否为数值）
   - **空值检查**：空值 >30% 警告并说明影响；关键字段（主键/时间/金额）空值 >5% 阻塞
   - **重复值检查**：主键重复阻塞；非主键重复说明去重规则
   - **异常值检查**：金额字段有负数需确认口径；时间字段超出合理范围需确认
4. 设计分析路径（根据业务问题选择）：
   - 问"总量多少、有多少" → **指标汇总**
   - 问"哪个维度最高、哪类占比大" → **分组对比**
   - 问"趋势如何、增长还是下降" → **趋势变化**
   - 问"为什么异常、为什么下滑" → **异常排查**（先看趋势，再拆维度，再看结构变化）
   - 问"结构是什么、分布如何" → **结构拆解**
5. 选择输出形式（根据用户信号判断）：
   - 用户问"结论是什么、有什么发现" → **直接输出**带口径说明的分析结论
   - 用户说"给我数据、列出来" → **Markdown 表格或 CSV**
   - 用户说"做个报告、给份材料、要转交" → **复用 Excel 生成 `.xlsx`**
6. 做结果校验：
   - 指标与筛选条件一致
   - 图表与表格一致
   - 结论与样本范围一致

## 门禁与降级
- 数据字段定义不清、主键不清或时间范围不清时，先停在口径确认，不直接给经营结论。
- 只有截图、零散文本或不完整表格时，先给可分析范围和缺失项，不伪造完整分析。

## Guardrails
- 不伪造数据、字段含义、筛选条件或 SQL 结果。
- 不把相关性直接写成因果。
- 不在未做数据体检时直接输出高置信度经营结论。
- 不为了“好看”而选择会误导的图表。
- 不复制 `Excel` 的底层产物逻辑；需要工作簿时复用现有 `Excel` 能力。
- 涉及代码、注释、文档或提交时，署名遵守全局规则：不用 AI，统一用 `anfeng`。

## Output
使用简体中文，先给结论：核心发现 → 分析口径 → 证据或表格/图表建议 → 风险与限制 → 下一步建议。
