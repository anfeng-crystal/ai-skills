# ISCB 上下文路由与输出契约

## 目录

- 1. 目的
- 2. 上下文决策表
- 3. 信息充分性检查
- 4. 输出契约
- 5. 校验与保存动作矩阵
- 6. 平台层降级模板

## 1. 目的

本文件是 `SKILL.md` 的外部权威参考，用来回答两类问题：

- 当前请求应该按哪种脚本上下文处理
- 校验、运行、保存到底在什么条件下才允许发生

如与其他文档冲突，以 `SKILL.md` 的主规则、validator 和 manifest 为准。

## 2. 上下文决策表

| 场景 | 典型信号 | 可以使用 | 不能默认假设 | 处理策略 |
|---|---|---|---|---|
| 普通引擎脚本 | “写个脚本”“帮我改脚本”“解释这段脚本” | 用户显式给出的变量名和对象结构 | `src`、`tar`、`param`、`cn`、`$process`、`#request` | 默认路径；优先生成最小必要脚本 |
| 数据集成方案 | “来源数据处理脚本”“转换脚本”“目标数据处理脚本” | `src`、`tar`、`$src`、`$tar` | 额外连接别名、更多平台对象结构 | 可使用数据集成上下文；若连接资源信息不足，仍需说明限制 |
| 值转换规则 | “脚本类型值转换规则” | `param`；在用户明确说明时可使用 `$src`、`$tar`、`$this` | `src` / `tar` 默认存在 | 优先保持小而纯的转换逻辑 |
| 服务流程脚本节点 | “服务流程”“流程脚本节点” | `$process` 和用户给出的资源别名 | 别名固定叫 `cn` | 生成时显式沿用用户给出的别名 |
| 自定义 API / 脚本 API | “自定义 API”“脚本 API” | 用户给出的 API 入参名 | `src` / `tar` 默认存在 | 若参数定义不全，先用参考脚本并标明需替换实际入参名 |
| WebAPI / 开放平台 | “WebAPI”“开放平台调用” | 用户已确认的请求变量；必要时 `#request` | 请求体字段名、响应对象结构 | 若缺结构说明，先给参考脚本并说明待补齐项 |

## 3. 信息充分性检查

进入平台层上下文前，至少检查以下信息是否充分：

| 场景 | 最低需要的信息 |
|---|---|
| 数据集成方案 | 来源/目标脚本位置，是否允许 `src` / `tar`，是否涉及连接资源 |
| 值转换规则 | 是否确实是脚本类型值转换规则，是否需要查库 |
| 服务流程脚本 | 节点类型，流程中可用资源别名 |
| 自定义 API | 实际参数名或最少的参数结构 |
| WebAPI | 可用请求变量、请求体结构或字段名 |

若这些信息不足：

- 允许生成参考脚本
- 必须显式标明“当前缺少哪些资源/上下文”
- 不得宣称“已验证可运行”
- 如提到本地静态校验结论，还必须区分“脚本缺平台上下文”和“本地工具 / manifest 只做弱预检”这两类原因

## 4. 输出契约

| 状态 | 触发条件 | 必须包含的信息 |
|---|---|---|
| `未校验` | 用户只要求生成、解释或重构 | 结果内容本身；明确未执行验证 |
| `已静态校验` | 用户明确要求静态校验 | 使用了哪种校验；关键 findings；是否通过 |
| `已编译校验` | 用户明确要求编译校验 | 真实 runtime compile 是否通过 |
| `已运行验证` | 用户明确要求运行验证 | 实际 `return_value` 或 `runtime_error` 关键内容 |

同一轮回答里，状态必须准确，不要混说。

## 5. 校验与保存动作矩阵

| 用户表达 | 允许动作 | 说明 |
|---|---|---|
| “帮我生成脚本” | 只生成 | 不自动编译、不自动运行、不自动保存 |
| “做静态校验” | 静态校验 | 不能声称真实 runtime 已通过 |
| “做编译校验” | 真实 runtime compile | 不能声称已运行 |
| “做运行验证” | 真实运行 | 必须回传实际结果或报错 |
| “保存到本地” | 写入文件 | 如只给目录，默认 `generated-script.iscb` |

## 6. 平台层降级模板

当平台资源不完整时，输出应遵循下面的语义：

- 先给参考脚本
- 再明确指出当前缺少的资源、别名、参数定义或对象结构
- 最后说明“需在苍穹集成平台补齐并验证”
- 若同时提到本地 `check-script` 结果，要额外说明这可能只是工具 / manifest 对平台能力的覆盖边界，不等于脚本语法错误

推荐表达：

```text
以下先给参考脚本。
当前缺少：实际资源别名 / API 参数定义 / 请求对象结构。
需要到苍穹集成平台补齐后再验证。
若本地 `check-script` 只给出弱预检或提示不在 engine-only 范围内，表示工具 / manifest 对平台能力是部分覆盖，不等于脚本语法错误。
```
