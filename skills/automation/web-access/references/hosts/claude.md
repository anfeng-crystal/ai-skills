# Claude 宿主适配

Claude 类宿主中，不要默认假设“搜索一定可用且稳定”。如果当前 Claude Code 通过中转站运行，默认搜索/抓取能力可能不可用、结果弱或不可预测，此时要优先走可验证的来源路径，而不是卡在宿主搜索上。

## 推荐顺序
1. 直接来源 URL、官方站点入口页、已知站内路径
2. 如果需要可编程搜索后端，且 `BRAVE_SEARCH_API_KEY` 已配置：`scripts/brave-search.mjs`
3. 宿主原生浏览器或桌面能力
4. 本 skill 的本地脚本：`check-deps.mjs`、`find-url.mjs`、`cdp-proxy.mjs`
5. `playwright`，仅在需要稳定重复抓取或批量页面操作时启用
6. 只有当前 Claude 环境明确具备可用搜索时，才把宿主搜索作为加速手段

## 使用原则
- 如果已经知道直接来源或站点入口，不要先去试宿主搜索。
- 如果 Claude 搜索不稳且又需要广泛发现候选来源，优先考虑 Brave 后端，而不是卡在宿主搜索上。
- 如果宿主已经能稳定读取网页内容，不要为了统一而强行改走 CDP。
- 如果任务依赖用户现有浏览器状态、登录态标签页或原始 DevTools 协议，再升级到本 skill 脚本。
- 如果任务需要重复翻页、滚动加载、结构化导出，优先切 `playwright`，不要用单次 CDP 命令硬拼流程。
- 宿主缺少浏览器工具时，可以继续做来源矩阵和访问方案；只有明确需要真实页面状态时才要求本机打开 CDP。

## 典型路由
- 最新事实核验：
  先直接来源和已知入口；需要可编程搜索时优先 Brave 后端；再看当前 Claude 是否能稳定搜索；不稳定就改走浏览器访问。
- 登录态页面：
  先让用户在浏览器完成登录，再用 `find-url.mjs` / `cdp-proxy.mjs` 接入现有 tab。
- 动态 SPA：
  先浏览器观察，再 CDP 单次补采；需要连续稳定抓取时切 `playwright`。

## 输出要求
- 明确说明这次有没有使用 Claude 宿主搜索。
- 明确说明这次有没有使用 Brave 搜索后端。
- 如果没有使用宿主搜索，要写明是因为宿主搜索不可用、不稳定，还是已有更直接来源。
