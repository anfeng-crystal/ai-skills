# Auth / Session Pattern

## 适用信号
- 页面依赖用户已登录状态
- 请求结果受租户、账号、权限、组织、语言环境或当前浏览器会话影响
- 页面内容公开地址可见，但真正数据只有登录后才出现

## 首选入口
- 优先复用用户已经登录的浏览器会话
- 先走宿主浏览器/电脑操作完成登录和人工确认
- 再用 CDP 做只读补采；不要让脚本替代登录流程

## 最小步骤
1. 确认用户已在本地浏览器登录成功。
2. `check-deps.mjs` 预检当前宿主与本地能力。
3. `cdp-proxy.mjs probe` 检查端点；`find-url.mjs` 找到正确 tab。
4. 先抓协议元信息、DOM 或快照；必须执行页面 JS 时，再显式 `--allow-unsafe`。
5. 输出时只保留必要字段，不回传敏感凭据。

## 证据记录
- 使用了哪个宿主入口
- 是否确认用户已登录
- 读取的是哪个页面 URL / target
- 页面事实、抓取时间和未确认项

## 风险 / 停问
- 未确认登录状态，不继续假设页面可读
- 页面含敏感数据时，先缩小提取范围
- 涉及提交、删除、发消息、付款等写操作时，默认停问
