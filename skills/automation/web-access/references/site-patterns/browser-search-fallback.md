# Browser Search Fallback Pattern

## 适用信号
- 当前宿主的默认搜索不可用、质量差、结果不稳定，或受中转站/权限限制
- 已知目标站点、官方来源或产品入口，但搜索入口不可靠
- 用户要的是“尽快拿到真实来源”，而不是先看聚合搜索结果

## 首选入口
- 优先直接来源 URL、官方导航页、帮助中心、产品文档首页、公告页
- 再用宿主浏览器访问页面，并在页面内搜索或站内跳转
- 必要时再用 CDP 或 `playwright` 补采，不把“先搜索一下”作为任务前提

## 最小步骤
1. 先列出最可能的直接来源：官网、文档、公告、帮助中心、产品后台入口。
2. 如果已知域名或入口路径，直接打开，不先依赖宿主搜索。
3. 页面能正常访问时，优先用页面导航、站内搜索、页内查找和链接跳转定位目标内容。
4. 需要复用现有登录态或读取已有标签页时，用 `find-url.mjs` / `cdp-proxy.mjs`。
5. 需要多页重复抓取、翻页或批量导出时，切 `playwright`。

## 证据记录
- 这次是否使用宿主搜索
- 如果没用，原因是“不可用”“不稳定”还是“已有直接来源”
- 最终依赖的是哪个来源入口
- 页面访问时间和仍未确认项

## 风险 / 停问
- 已知入口不等于最终事实页，仍需核对页面发布日期和正文内容
- 搜索不可用时，更不能把记忆当现况
- 如果站点必须登录或有人机校验，先让用户接管敏感步骤
