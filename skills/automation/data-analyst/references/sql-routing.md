# SQL Routing

## 什么时候先看 SQL
- 用户直接给了查询语句
- 需要从数据库取数
- 指标口径依赖过滤条件、关联方式或时间窗口

## 默认顺序
1. 先确认表、主键、时间字段和指标字段
2. 再确认筛选条件：
   - 时间范围
   - 状态过滤
   - 去重规则
   - 是否排除测试数据
3. 再决定查询形态：
   - 明细抽样
   - 聚合汇总
   - 分组对比
   - 时间趋势

## 常见风险
- 把订单数和订单行数混为一谈
- 关联后重复放大金额
- 用创建时间和业务时间混算
- 忽略取消、退款、作废等状态
- 把空值当 0 或把 0 当空值

## 输出要求
- 解释关键过滤条件
- 说明分组维度
- 说明是否做了去重
- 说明金额、数量、人数这些指标的口径差异
