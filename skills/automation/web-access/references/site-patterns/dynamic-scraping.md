# Dynamic Scraping Pattern

## 适用信号
- 任务不是一次性查看，而是要重复抓取、翻页、滚动加载或批量导出
- 同一套页面操作需要在多个 URL、多个 tab 或多轮结果页上重复执行
- 单次宿主浏览器操作或单条 CDP 命令已经明显不够稳定

## 首选入口
- 先用宿主浏览器的单步动作循环或 CDP 探明页面结构、登录态和关键交互
- 确认流程会重复后，优先切 `playwright`
- 只有必须复用用户当前 tab 或已登录会话时，保留 CDP 作为补充

## 最小步骤
1. 先确认抓取目标：字段、页数、分页方式、导出格式和停止条件。
2. 用宿主浏览器或 CDP 探清楚页面是否依赖登录态、滚动、懒加载、切 tab、文件下载。
3. 把“可重复执行”的流程交给 `playwright`：打开、滚动、点击、翻页、提取、保存。
4. 抓取过程中持续记录来源 URL、时间、分页位置和失败页。
5. 输出时区分：探索阶段证据、批量抓取结果、未覆盖页和失败页。

## 证据记录
- 抓取入口和字段定义
- 是否需要登录态
- 分页/滚动/加载更多的触发条件
- 已成功页数、失败页数、终止条件

## 风险 / 停问
- 没有明确字段和终止条件时，不要直接开始批量抓取
- 页面结构频繁变化时，先确认是否仍值得自动化
- 如果站点有明显反爬、验证码、下载限流或企业安全策略，先暂停并和用户确认
