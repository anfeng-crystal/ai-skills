# ISCB 资源类型与上下文变量

> 本文件用于**确认用户已经给出的脚本上下文**，不是用来替用户猜上下文的。
> 用户没有明确说明资源/节点类型时，不要因为这里列出了变量，就默认生成 `src`、`tar`、`param`、`cn`、`$process`、`#request` 等写法。

## 资源类型目录

脚本运行在不同的资源上下文中，每种资源提供不同的系统变量和能力。

| 资源类型 | 标识 | 说明 |
|----------|------|------|
| 数据源 | ISC_DATA_SOURCE | 外部数据源连接 |
| 数据集成方案 | ISC_DATA_COPY | 数据映射/转换方案 |
| 值转换规则 | ISC_VALUE_CONVER_RULE | 字段值转换规则 |
| 数据集成触发器 | ISC_DATA_COPY_TRIGGER | 数据集成事件触发 |
| 自定义函数 | ISC_CUSTOM_FUNCTION | 用户自定义脚本函数 |
| 元数据方案 | ISC_METADATA_SCHEMA | 元数据定义方案 |
| 外部API | ISC_APIC_FOR_EXTERNAL_API | 外部 API 配置 |
| 脚本API | ISC_APIC_SCRIPT | 脚本定义的 API |
| 消息队列订阅 | ISC_MQ_SUBSCRIBER | MQ 消息订阅处理 |
| 服务流程 | ISC_SERVICE_FLOW | 服务编排流程 |
| 微服务 | ISC_APIC_MSERVICE | 微服务引用 |
| 消息队列发布 | ISC_MQ_PUBLISHER | MQ 消息发布 |
| 数据组件 | ISC_DATA_COMP | 数据组合组件 |
| 文件导出触发器 | ISC_EXPORT_FILE_TRIGGER | 文件导出事件触发 |
| 文件导入触发器 | ISC_IMPORT_FILE_TRIGGER | 文件导入事件触发 |
| 用户自定义事件 | ISC_USER_DEFINED_EVENT | 自定义业务事件 |
| WebAPI | ISC_APIC_WEBAPI | WebAPI 登记 |
| 数据流触发器 | ISCX_DATA_FLOW_TRIGGER | 数据流事件触发 |

## 上下文变量可用性

### 数据集成方案：来源数据处理脚本
| 变量 | 类型 | 说明 |
|------|------|------|
| `src` | 对象/Map | 源单当前行预置变量，可直接用 `src.field` 访问 |
| `$src` | 连接 | 源系统数据库连接 |

**常见用途**：字段映射前清洗/补充源数据、打 `#_IGNORE_TAG`

**特殊限制**：
- 这里默认只说明帮助库中有明确依据的变量；不要自动假设存在 `tar`
- XPath 访问语法另行使用 `@.`、`@../var`、`@/var`、`@variable/path`，不要写成 `@src.field`

### 数据集成方案：转换脚本
| 变量 | 类型 | 说明 |
|------|------|------|
| `src` | 对象/Map | 源单当前行预置变量 |
| `tar` | 对象/Map | 目标单当前行预置变量 |
| `$src` | 连接 | 源系统数据库连接 |
| `$tar` | 连接 | 目标系统数据库连接 |
| `$this` | 连接 | 当前苍穹环境数据库连接 |
| `@@var` | 上下文变量 | 嵌入式 SQL 变量绑定 |

**常见用途**：字段映射后补充目标数据、跨系统查库、过滤目标数据

### 数据集成方案：目标数据处理脚本
| 变量 | 类型 | 说明 |
|------|------|------|
| `tar` | 对象/Map | 目标单当前行预置变量 |
| `$tar` | 连接 | 目标系统数据库连接 |

**常见用途**：保存前后补充处理、反写目标系统、决定最终保存结果

### 脚本类型值转换规则
| 变量 | 类型 | 说明 |
|------|------|------|
| `param` | Object | 固定输入参数，脚本型值转换规则优先使用它 |
| `$src` | 连接 | 源系统数据库连接 |
| `$tar` | 连接 | 目标系统数据库连接 |
| `$this` | 连接 | 当前苍穹环境数据库连接 |
| `@@var` | 上下文变量 | 嵌入式 SQL 变量绑定 |

**特殊限制**：
- `值转换规则` 不要默认写成 `src` / `tar` 风格；除非用户明确说脚本写在集成方案转换节点里
- 脚本型值转换规则的固定输入通常是 `param`

### 服务流程上下文
| 变量 | 类型 | 说明 |
|------|------|------|
| `$process` | ServiceFlowProcess | 当前流程实例 |
| `资源别名` | 连接/资源 | 通过流程属性 -> 依赖资源引入后，用**资源别名**访问；帮助库示例常用 `cn`，但不是固定变量名 |

**可用函数**：全部 SQL 函数（包括 execute_update/execute_batch）、HTTP 函数、所有平台层函数

### 自定义 API / 脚本 API 上下文
| 变量 | 类型 | 说明 |
|------|------|------|
| `API 定义的输入参数` | 上下文变量 | 参数名取决于 API 定义，不是固定变量名；若未提供定义，不要臆造 |
| `资源别名` | 连接/资源 | 若脚本资源中引入了数据源或其他资源，应按资源别名访问；不要默认固定为 `cn` |
| `#request` | 请求对象 | 仅当自定义 API 通过开放平台调用时，可读取 headers、queryString、url、data |

**可用函数**：全部 SQL 函数（包括 execute_update/execute_batch）、HTTP 函数、所有平台层函数

### WebAPI 上下文
| 变量 | 类型 | 说明 |
|------|------|------|
| `$cn` | 连接 | WebAPI 连接 |

**说明**：
- 帮助库当前未提供一组已核实的、固定命名的 WebAPI 请求/响应上下文变量
- 因此这里不要笼统写成“请求/响应上下文”，更不要臆造变量名
- 如果用户明确说脚本写在 WebAPI 资源中，应优先结合现有脚本、运行时返回对象或平台实际配置确认可用变量名

## 连接变量使用规则

1. **$src / $tar / $this**：只在数据集成方案或脚本类型值转换规则等特定上下文中可用
2. **$cn**：在使用了连接器的资源中可用（FTP、HTTP、WebAPI 等）
3. **`cn`（无 $ 前缀）**：帮助库中常被用作资源别名示例，但不是通用固定预置变量；应以实际资源别名为准
4. **$process**：只在服务流程中可用

> **重要**：只有当用户明确说明脚本上下文时，才使用该上下文的预置变量。用户只说“写个脚本”时，默认不要假设存在 `src`、`tar`、`param`、`cn`、`$process`、`#request` 等变量。
