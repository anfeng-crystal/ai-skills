# 分析口径

## 实体层
- 说明实体标识、实体名称、页面数量、插件数量和源码可见性。
- 区分操作插件、页面插件、标准平台插件和项目自定义插件。

## 插件层
| 维度 | 要求 |
|---|---|
| 基本信息 | 类名、插件类型、绑定页面或操作、启用状态、源码来源 |
| 继承关系 | 识别基类和关键接口 |
| 关键方法 | 提炼重写方法、调用入口、条件分支和失败边界 |
| 字段读写 | 列出读取字段、写入字段、字段来源和字段用途 |
| 服务调用 | 列出 ServiceHelper、QueryServiceHelper、OperationServiceHelper 等调用 |
| 外部实体 | 列出查询、保存、提交、审核、回写涉及的其他实体标识 |
| 复用机会 | 标出已有插件源码、模板、helper、标准服务或平台能力，说明哪些后续需求可直接复用，避免重复实现 |

## 平台标准类
- `kd.bos.*` 标准类只说明平台用途，不展开反编译实现。
- 找不到源码时，以 `javap` 签名、类名、绑定位置和平台命名推断用途，并标注待确认。

## 风险边界
- 区分确定事实、源码推断和待确认项。
- 涉及跨库保存、直接 SQL、全量查询、报表写库、DataSet 生命周期时单独标注风险。
- 不把命令执行过程、修复经过或交付说明写入分析报告。
