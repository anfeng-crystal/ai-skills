# 脚本配置规则

## 使用时机
- 只有字段、API、事件、实体标识或基础资料标识需要脚本确认时，才执行查询脚本。
- 当前项目已经能通过源码、依赖或官方资料确认时，不强制执行脚本。

## 配置要求
- 调用脚本时优先显式传入 `--config <配置文件>`。
- 配置文件不得写入真实令牌、密码或个人机器专属路径。
- 配置缺失时，说明缺口并改用项目源码、依赖、官方资料或知识库交叉校验。

## 查询顺序
1. 当前项目源码和元数据。
2. `${AI_KNOWLEDGE_ROOT}/kingdee/cosmic`。
3. 本 skill 的 `references/`、`references/rules/`、`assets/`。
4. 脚本查询结果。
5. 官方资料。

## 复用优先
- 写插件、helper、工具类或复杂方法前，先检查当前项目源码、模板、snippet、标准服务和已有公共封装是否已覆盖需求。
- 如果现成能力可用，优先复用或在原有扩展点补最小改动；不要再写第二套查询、回写、校验或平台包装逻辑。

## 事实确认
- 遇到 `entityId`、`formId`、`refType`、字段、事件点、API 签名不确定时，先读 `fact-confirmation.md`，按“关键事实停问 + 本地证据等级 + 在线增强兜底”判断是否需要停问。
- 在线元数据/基础资料查询只作为可用时增强；VPN 或代理导致不可用时，优先用本地源码、references 和脚本离线结果继续。
