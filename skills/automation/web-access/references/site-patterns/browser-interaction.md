# Browser Interaction Pattern

## 适用信号
- 任务需要真实页面交互，但还在探索阶段
- 需要点击、输入、展开、切换、等待加载、观察页面变化
- 用户要的是“先把流程走通”，而不是立即写成完整自动化

## 单步动作循环
1. 先观察当前页面状态：
   - 当前 URL
   - 关键区域
   - 可点击元素
   - 是否有弹窗、登录、验证码
2. 只做一个动作：
   - 点击
   - 输入
   - 选择
   - 滚动
   - 切 tab
3. 动作后立刻复核：
   - 页面是否变化
   - 目标元素是否出现
   - 是否进入下一步状态
4. 只有流程稳定重复时，才迁移到 `playwright`

## 选择原则
- 视觉确认、临时操作：宿主浏览器或桌面能力
- 已登录 tab、运行时值、DOM 补采：CDP
- 重复流程、批量页面、结构化采集：`playwright`

## 风险 / 停问
- 没看清页面状态前，不连做多步动作
- 遇到验证码、支付、发布、删除、发送等高风险动作时暂停
- 如果同一步骤在多个页面要反复执行，不要继续手工或单次 CDP 硬拼，直接升级到 `playwright`
