# Kingdee VIP / 金蝶云社区知识页经验

适用范围：
- 域名：`vip.kingdee.com`
- 典型路径：`/knowledge/<id>`
- 场景：已登录的 VIP 知识页、社区知识文章详情、登录态复用验证

## 已验证事实
- 在独立 Chrome CDP 实例中，已登录状态下可以直接访问知识文章详情页。
- 对当前已打开的知识页标签页做只读验证可成功读取：
  - 页面标题
  - URL
  - 正文文本长度
  - 正文关键词（如“苍穹”“单点登录”）
- 在同一个已登录的 Chrome 实例里，新开后台 tab 访问同一知识页，页面加载完成后可继承登录态进入知识页，不会强制跳回登录页。

## 推荐执行面
1. 优先：CDP 复用现有已登录标签页，先做只读验证
2. 次选：在同一 Chrome 实例里新开 tab，验证是否继承登录态
3. 最后：如果高层脚本超时，再降级到原始 WebSocket CDP 调用

## 稳定定位线索
- 标题常见命中：`轻松搞定苍穹单点登录`
- URL 可直接按完整知识页链接匹配
- 页面正文常可命中：
  - `金蝶云社区`
  - `苍穹平台那些事`
  - `登录认证`
  - 文章标题关键字

## 已知陷阱
- 某些页面上，`cdp-proxy.mjs` 的高层命令（如 `info` / `screenshot` / `send`）可能超时，但页面级原始 WebSocket CDP 调用依然可成功返回。
- 使用 Chrome 147 时，`/json/new` 打开新标签页应使用 `PUT` 而不是 `GET`，否则会返回：
  - `Using unsafe HTTP verb GET to invoke /json/new. This action supports only PUT verb.`
- 新开 tab 后，初始瞬间可能先停留在 `about:blank`，应等待页面真实加载完成后再判断登录态或内容状态。

## 推荐验证顺序
1. `check-deps.mjs --json`
2. `find-url.mjs` 精确定位现有知识页 tab
3. 先做只读验证（标题 / URL / 文本长度 / 关键词）
4. 如需验证继承登录态，再新开 tab
5. 若高层命令超时，回退到原始 WebSocket CDP

## 安全边界
- 默认只做只读验证和轻交互（如滚动）
- 不自动执行发布、下载、账号设置修改等动作
- 若页面需要进一步状态变更，先向用户解释目标动作与风险
