# Validation

## 数据体检标准

### 空值检查
- **空值 >30%** → 警告并说明影响（例如："地区字段空值 35%，区域分析结论仅覆盖 65% 样本"）
- **关键字段（主键/时间/金额）空值 >5%** → 阻塞，必须先清洗或说明处理规则

### 重复值检查
- **主键重复** → 阻塞，必须先去重或说明去重规则
- **非主键重复** → 说明去重规则（例如："同一用户多次购买，按订单数统计"）

### 异常值检查
- **金额字段有负数** → 需确认口径（退款/调整/错误数据）
- **时间字段超出合理范围** → 需确认（例如：订单时间在 2050 年）
- **极端值** → 单独说明（例如："最大值 1000 万，为异常订单，已排除"）

### 字段类型检查
- **日期字段是否可解析**：格式不一致 → 需统一格式
- **金额字段是否为数值**：包含文本 → 需清洗
- **分类字段是否有脏值**：例如"北京"和"北京市"、"Male"和"male" → 需合并口径

## 最小校验清单
- 时间范围是否写清
- 样本范围是否写清
- 主键或去重规则是否写清
- 指标公式是否能复算
- 图表与表格是否一致
- 结论是否超出数据覆盖范围

## 结论检查
- 有没有把相关性写成因果
- 有没有忽略样本太小的问题
- 有没有把未覆盖月份或未覆盖渠道说成全量结论
- 有没有把局部异常当成整体趋势

## 文件交付检查
- 用户明确要工作簿时，再转入 `Excel`
- sheet 名、图表名和口径说明是否一致
- 交付件里是否保留了必要的来源和限制说明
