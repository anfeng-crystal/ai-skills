# Chart Playbook

## 选择原则
- 图表要服务于问题，不是为了装饰。
- 同一个结论优先选最容易读懂的图，不追求复杂样式。

## 业务问题到图表类型的映射

### 趋势类问题
- 问"趋势如何、增长还是下降、最近几个月变化" → **折线图**
- 多条趋势对比 → **多条折线图**（不超过 5 条线）

### 对比类问题
- 问"哪个维度最高、哪类占比大、排名如何" → **条形图**（降序排列）
- 时间段对比（例如：各月销售额） → **柱状图**

### 结构类问题
- 问"结构是什么、占比分布" → **条形图优先**（显示具体数值和占比）
- 类别很少（≤5 个）且强调整体占比 → **饼图**（标注百分比）

### 分布类问题
- 问"分布如何、集中在哪个区间" → **直方图或分箱条形图**

### 转化类问题
- 问"转化率、漏斗、流失在哪" → **漏斗图或分阶段条形图**

## 避免误导
- 非零基线柱状图要谨慎（容易夸大差异）
- 类别过多（>10 个）不要硬塞饼图（改用条形图 + Top N）
- 轴标签过密时先聚合或缩短范围
- 同时展示金额和比例时，优先拆成两张图或明确双轴风险

## 交付建议
- 经营汇报常用：
  - KPI 摘要
  - 趋势图
  - 结构对比图
- 明细排查常用：
  - 排名条形图
  - 异常点标注
  - 分组透视表
